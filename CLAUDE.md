## Что за проект

Полноценный **антивирус для Windows**, строится поэтапно. Цель — не учебная поделка, а
**настоящий, качественный, рабочий антивирус** (пока для личного использования, в перспективе —
для раздачи близким). Автор учится по ходу, но планка — реальный продукт, а не демо.
Начинали с CLI-сканера по VirusTotal; сейчас это уже **мульти-источниковый сканер +
GUI-приложение в стиле кибер-терминала + фоновая защита**. Текущая версия — **v0.23**.

> Из этого следует планка качества: код и решения оцениваем как для боевого продукта —
> надёжность, обработка ошибок, реальные тесты на Windows, а не «лишь бы запустилось».

## Принятые решения (важно, не пересматривать без причины)

- **Язык — Python.** Вся логика на Python. C/C++ появится только на этапе 4 (драйвер ядра,
  minifilter) — отдельный модуль-«датчик», Python остаётся «мозгом». До этапа 4 C++ не трогаем.
- **Проверка по SHA-256, не загрузка файла** — по умолчанию шлём только хэш. Загрузка файла
  на анализ — по флагу `--upload`.
- **Безопасность ключей** — ключ VirusTotal в `.vtkey`, ключи доп. источников в `keys.json`.
  Оба в папке данных пользователя, в репозиторий/в exe НЕ попадают. Никогда не коммитить ключи.
- **КРИТЕРИЙ «одна папка»** — всё (ключи, ClamAV, карантин, белый список) в ОДНОМ месте.
  Собранный exe ВСЕГДА хранит данные в `%LOCALAPPDATA%\VTScan` (стабильно, переживает
  переустановку/обновление). В dev-режиме (запуск .py) — рядом со скриптом. См. `engines.data_dir()`.
- **Два exe** (с этапа 3): `VTScan.exe` — GUI-окно (pywebview, основной продукт),
  `vtscan-cli.exe` — командная строка. Имена обязаны отличаться не только регистром
  (ФС Windows регистронезависима).
- **Реакция фоновой защиты — обратимая по умолчанию:** карантин файла (шифруется при помещении)
  и пауза процесса (suspend), а НЕ удаление/kill. Жёсткое — по кнопке/подтверждению.
- **Деньги/подписи:** этапы 1–3 (user-mode) — БЕСПЛАТНО, подпись не нужна (только косметическое
  предупреждение SmartScreen). Платное — только этап 4 (драйвер) и только для раздачи чужим.
- **Уровни прав ≠ этапы фич (не путать).** «Этапы» — это функциональность (1 VT, 2 мульти-источник,
  3 GUI+фоновая защита, 4 драйвер). Отдельно есть «уровни прав» — насколько глубоко антивирус
  вкопан в систему: (1) обычный user-mode [сейчас]; (2) администратор/UAC; (3) служба SYSTEM/PPL;
  (4) ядро (Ring 0, драйвер). Деньги — НЕ блокер для ЛИЧНОГО антивируса: на своём ПК уровни 3–4
  подписываются бесплатно через **test-signing** (самоподписанный драйвер/ELAM). Платно — только
  для РАЗДАЧИ другим (EV-сертификат + аттестация Microsoft). Реальная цена уровней 3–4 — не деньги,
  а труд (C/C++ ядро = этап 4) и побочки test-signing (грузит любой самоподписанный драйвер,
  водяной знак, конфликты Secure Boot — тестировать в виртуалке). Целевой потолок для себя:
  уровень 2 (админ) как ближайшая user-mode задача, уровни 3–4 — в рамках этапа 4.

## Текущее состояние

- **Этап 1 (готов):** SHA-256 → VirusTotal, вердикт + тип угрозы + семейство.
- **Этап 2 (готов):** мульти-источник — общий интерфейс «движок» + агрегатор:
  VirusTotal + ClamAV (офлайн, авто-загрузка в папку) + MalwareBazaar/OTX/Kaspersky
  (по ключу) + белый список по SHA-256.
- **Этап 3 (в работе):** GUI-терминал (pywebview), фоновая защита (`monitor`): монитор
  автозагрузки + процессов (только подозрительные) + новые файлы → карантин; **приманки-ловушки
  против шифровальщиков** (canary) + заморозка подозрительных; системные уведомления с кнопками
  выбора; автодополнение команд; авто-обновление; установщик `VTScan-Setup.exe` (Inno Setup).
- **Этап 4 (на потом):** драйвер ядра C/C++ (см. backlog).

## Архитектура (файлы в `apps/vtscan/`)

- `vtscan.py` — ядро + CLI + интерактивный терминал. Сканер, VirusTotalClient, агрегатор,
  команды (`scan/monitor/keys/quarantine/check-update/setup-clamav/selftest/...`), авто-обновление.
- `engines.py` — общий контракт `EngineResult`/`aggregate_status`, `data_dir()`, движок ClamAV
  и его авто-загрузчик (`provision_clamav`), белый список.
- `sources.py` — доп. онлайн-источники по хэшу (MalwareBazaar, AlienVault OTX,
  Kaspersky OpenTIP); ключи в `keys.json`; источник без ключа пропускается.
- `monitor.py` — фоновая защита: снимки автозагрузки/процессов, watchdog по папкам, карантин
  (с шифрованием), приманки-шифровальщик, заморозка процессов, системные уведомления (`toast`).
- `gui.py` — окно-приложение (pywebview) в стиле кибер-терминала; переиспользует весь движок.
- `build_windows.bat` — локальная сборка (запасная); основная — в облаке (см. ниже).
- `apps/done/` — готовые exe (не в git, синкаются через SynologyDrive).
- `apps/test-samples/` — безвредные EICAR для теста (eicar-файлы не в git).
- `installer/VTScan.iss` — Inno Setup скрипт установщика.

## Сборка и релизы (GitHub Actions)

- Репозиторий: https://github.com/SergeySmirnovGitHub/antivirus-project (публичный).
- `.github/workflows/build-windows.yml` собирает оба exe + установщик на `windows-latest`.
- **Релиз:** поднять `VERSION` в `vtscan.py` → коммит → `git push` → `git tag vX.Y && git push origin vX.Y`.
  По тегу собирается Release с `VTScan.exe`, `vtscan-cli.exe`, `VTScan-Setup.exe`.
- Скачать свежие: `gh run download <id> -n vtscan-windows -D apps/done` (перед этим `rm -f apps/done/*.exe`).
- `gh` (CLI) на dev-машине лежит в `~/.local/bin/gh`; сетевые git/gh-команды требуют
  `dangerouslyDisableSandbox`. Детали — в памяти `github-ci-setup`.

## Перенос на другую машину

Код едет через GitHub. На новой машине:
1. `git clone https://github.com/SergeySmirnovGitHub/antivirus-project`
2. `python3 -m venv .venv && .venv/bin/pip install -r requirements.txt`
3. Положить свой ключ VirusTotal в `apps/vtscan/.vtkey` (см. `.vtkey.example`).
4. Для релизов: установить `gh` и `gh auth login` (нужны scopes `repo, workflow`).
5. Запуск: `.venv/bin/python apps/vtscan/vtscan.py -i` (CLI) или `gui.py` (окно).

Не в git (создаётся заново): `.venv/`, `.vtkey`, ключи источников, собранные exe, `.claude/`.
