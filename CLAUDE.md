# CLAUDE.md — контекст проекта для Claude Code

> Этот файл Claude Code читает автоматически при запуске в папке проекта.
> Здесь — вся история и договорённости, чтобы не объяснять заново.

## Что за проект

Учебный антивирус для Windows, который строится поэтапно. Автор — начинающий, цель
учиться по ходу дела. Начали с самого простого и полезного: CLI-сканер файлов
через VirusTotal API. Дальше — постепенно к фоновой защите.

## Принятые решения (важно, не пересматривать без причины)

- **Язык — Python.** Вся логика (сканер, эвристика, интерфейс, фоновый сервис)
  остаётся на Python. Это осознанный выбор ради скорости разработки.
- **C/C++ появится только на этапе 4** — и только для драйвера ядра (minifilter),
  потому что иначе реал-тайм-перехват файлов в Windows невозможен. Это будет
  ОТДЕЛЬНЫЙ маленький модуль-«датчик», а не переписывание проекта. Питон-часть
  будет общаться с драйвером, а не заменяться им. До этапа 4 C++ не трогаем.
- **Проверка по SHA-256, а не загрузка файлов** — по умолчанию шлём в VirusTotal
  только хэш (быстро и приватно). Загрузка самого файла — только по флагу `--upload`.
- **Безопасность ключа** — API-ключ лежит в `.vtkey`, который в `.gitignore`.
  Никогда не коммитить ключ и не писать его в код.
- **КРИТЕРИЙ «одна папка»** — всё работает из одной папки с программой. Пользователь
  скачивает один наш продукт — и он работает, без ручной установки сторонних
  антивирусов. Локальные движки (ClamAV и т.п.) и их базы **бундлятся/авто-провижнятся
  в подпапку рядом с exe** (`<папка_exe>/engines/...`), не разбросаны по системе.
  Движок ищет свой бинарь в папке приложения в первую очередь. На этапе инсталлятора —
  отдельная папка под антивирус. (См. память `single-folder-constraint`.)

## Текущее состояние (этап 1 — ГОТОВО)

`apps/vtscan/vtscan.py` — рабочий CLI-сканер:
- считает SHA-256 файла или всех файлов в папке (с `-r` рекурсивно);
- запрашивает VirusTotal API v3 по хэшу, разбирает `last_analysis_stats`;
- по `--upload` загружает неизвестные файлы на анализ и ждёт результат;
- вывод в человекочитаемом виде или `--json`;
- учитывает лимит бесплатного ключа (4 запроса/мин → пауза 16 сек, флаг `--delay`);
- код возврата 1, если найден вредоносный файл.

Запуск: `python apps/vtscan/vtscan.py <файл-или-папка> [-r] [--upload] [--json]`
Ключ: положить в `apps/vtscan/.vtkey` (см. `apps/vtscan/.vtkey.example`).

- `apps/` — рабочие программы и прототипы, каждая в своей подпапке (`apps/vtscan/` — этап 1).

## Roadmap (следующие шаги)

### Этап 2 — локальный движок ClamAV (офлайн-проверка)
- Подключить ClamAV (open-source антивирус с бесплатной обновляемой базой сигнатур).
- На Windows: установить ClamAV, запустить демон `clamd`, ходить в него по сокету
  (библиотека `clamd`/`pyclamd`) ИЛИ вызывать `clamscan` как процесс.
- `freshclam` обновляет базу сигнатур.
- Цель: сканер работает без интернета и не упирается в лимиты VirusTotal.
- Архитектура: общий интерфейс «движок проверки», VirusTotal и ClamAV — две реализации;
  итоговый вердикт агрегирует оба источника.

### Этап 3 — фоновое наблюдение (чистый Python, без драйвера)
- `watchdog` — следить за появлением новых файлов в Downloads и др. папках,
  автоматически прогонять их через сканер этапов 1–2.
- `psutil` — мониторинг запущенных процессов (CPU, сеть, родитель, путь к exe).
- Простая поведенческая эвристика: подозрительные признаки → score → уведомление.
- Возможность приостановить (suspend) подозрительный процесс из user-mode.
- Упаковать как фоновый сервис/трей-приложение.

### Этап 4 — резидентная реал-тайм защита (продвинутое, на потом)
- Minifilter-драйвер ядра на C/C++ через Windows Driver Kit (WDK) — перехват
  открытия файлов в реальном времени.
- Требует EV-сертификата для подписи драйвера (платно) и регистрации у Microsoft.
- Драйвер шлёт события в Python-сервис, тот принимает решение.
- Это большой отдельный подпроект; браться, только когда этапы 1–3 устойчивы.
