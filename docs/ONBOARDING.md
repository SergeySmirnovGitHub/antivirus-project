# Онбординг проекта — быстрый старт для нового участника

> Этот документ — чтобы новый человек (или новая сессия Claude) за 5 минут понял,
> что это за проект, где что лежит и как всё собирается. Подробности — в файлах,
> на которые тут ссылки.

## 1. Что это

Учебный **антивирус для Windows**, строится поэтапно. Автор — начинающий, учится по ходу.
Логика на **Python**. C/C++ появится только на этапе 4 (драйвер ядра).

Главный файл контекста для Claude Code — **`CLAUDE.md`** в корне (читается автоматически).
Полный список задач по этапам — **`backlog.md`**.

## 2. Текущее состояние

- **Этап 1 (готов):** CLI-сканер `apps/vtscan/vtscan.py` — считает SHA-256, проверяет файл
  через VirusTotal API v3, показывает вердикт + тип угрозы (троян/стилер/…) и семейство.
- **Этап 1.5 (готов):** интерактивный «кибер-терминал» (запуск без аргументов / двойной клик),
  команды `scan / help / key / cd / clear / version / exit`, цветной вывод (ANSI + colorama),
  prompt подстраивается под компьютер. Сборка в один `.exe`.
- **Дальше:** этап 2 — мульти-источник (ClamAV + YARA + открытые базы), агрегатор вердиктов.

## 3. Структура папок

```
antivirus-project/
├── apps/
│   ├── vtscan/        # исходники сканера (этап 1): vtscan.py, build_windows.bat, .vtkey.example
│   └── done/          # готовые сборки (vtscan.exe — НЕ в git, синкается через SynologyDrive)
├── docs/              # документация (этот файл)
├── .github/workflows/ # build-windows.yml — авто-сборка .exe в облаке
├── backlog.md         # задачи по этапам
├── CLAUDE.md          # контекст для Claude Code
└── requirements.txt   # зависимости Python (requests, colorama)
```

## 4. Как запускать (разработка на Mac)

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/python apps/vtscan/vtscan.py <файл>      # разовая проверка
.venv/bin/python apps/vtscan/vtscan.py -i          # интерактивный терминал
```

Ключ VirusTotal: положить в `apps/vtscan/.vtkey` (или переменная `VT_API_KEY`).
Тест без вреда — файл EICAR (строка в `CLAUDE.md`), детектится всеми движками.

## 5. Как собирается .exe (для Windows)

Два пути, оба дают один `vtscan.exe`:
1. **Облако (основной):** `git push` в `main` → GitHub Actions собирает на windows-сервере →
   готовый exe лежит как артефакт `vtscan-windows`. Скачать: со страницы Actions ИЛИ
   `gh run download <id> -n vtscan-windows -D apps/done`.
2. **Локально (запасной):** на Windows двойной клик по `apps/vtscan/build_windows.bat`
   (нужен Python с «Add Python to PATH»).

Подробности про репозиторий, аккаунт и где лежит `gh` — в памяти проекта (`github-ci-setup`).

## 6. Безопасность ключа (важно!)

- Ключ VirusTotal лежит ТОЛЬКО в `.vtkey`, который в `.gitignore` → **в git/GitHub не попадает**.
- В собранный `.exe` ключ тоже не вшивается — программа спрашивает его при первом запуске.
- Поэтому даже в публичном репозитории чужие люди НЕ получают наш ключ: каждый подключает свой.
- Правило: **никогда не коммитить `.vtkey`**. Перед публикацией проверять историю git на утечки.

## 7. Принятые решения (не пересматривать без причины)
